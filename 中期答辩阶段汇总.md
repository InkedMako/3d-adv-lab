## 4. 与开题目标的差距分析

### 4.1 目标完成度判定

1. 统一实验入口：已完成。
2. 多模态产物标准化：已完成。
3. 多模态基线能力验证：部分完成。
4. 攻防对比实验矩阵：部分完成。

### 4.2 主要差距（中期后需重点补齐）

1. 模型有效性差距
   - 当前官方 AP 为 0.0，说明还未形成可证明“检测有效”的基线能力。
   - 需要确保模型-权重-配置-数据标注对齐，得到非零且可信的基线指标。

2. 攻击有效性差距
   - 当前 adv 与 clean 基本无差，尚未体现攻击成功（ASR 或性能下降）。
   - 需要引入更有区分度的攻击设置与预算扫描。

3. 防御有效性差距
   - 当前 defended 与 adv 基本无差，尚未体现防御恢复能力。
   - 需要建立“攻击造成下降 + 防御部分恢复”的对照证据链。

4. 协议完整性差距
   - “点云单攻 / 图像单攻 / 联合攻击”三类协议虽已具备实现基础，但还需要系统化跑表与统一结论汇总。

## 6. 下一阶段计划（建议作为中期后工作包）

### 工作包 A：建立可用基线（优先级最高）

1. 校验模型权重与配置匹配关系，确保基线 AP 非零。
2. 在固定样本集上输出 clean 基线报告与可视化。

### 工作包 B：提升攻击可见度

1. 分别执行点云单攻、图像单攻、联合攻击。
2. 进行预算扫描（小/中/大）并统计 ASR 与 AP 降幅。

### 工作包 C：验证防御恢复能力

1. 在已验证有效攻击上应用输入端防御。
2. 输出恢复率指标：
   - 恢复率 = (defended - adversarial) / (clean - adversarial)

### 工作包 D：形成中后期答辩材料

1. 汇总对比表：clean / adv / defended 三分支核心指标。
2. 汇总典型样本图：成功攻击案例、成功防御案例、失败案例。
3. 整理可复现实验命令清单与版本矩阵。

## 7. 结论（中期答辩可直接陈述）

当前项目已经完成“多模态鲁棒性实验平台”的工程闭环建设，并实现了 20 样本的正式评估与可视化产出；官方 KITTI 评估链路也已可运行。

与开题目标相比，下一阶段核心任务不再是“能不能跑通”，而是“让指标体现有效差异”：

1. 得到可信的非零基线检测能力；
2. 证明攻击能显著降低性能；
3. 证明防御能部分恢复性能；
4. 在点云单攻、图像单攻、联合攻击三种协议下形成完整可复现结论。
