# 测试数据集、攻击、防御与可视化工作流

本文说明在当前 ART + MMDetection3D 环境下，应该如何配置测试数据集、导入数据、生成攻击样本、防御样本和干净样本，以及如何得到可视化结果。

## 1. 先明确你的实验对象

建议先不要同时做太多任务，优先选一种：

- 3D 目标检测：推荐 KITTI 或 nuScenes
- 点云分类：推荐 ModelNet40 或 ScanObjectNN

如果导师当前更强调多模态，优先选：

- nuScenes
- LiDAR + Camera 融合模型

如果你们想先把“点云攻击与防御”跑通，先从：

- ModelNet40
- 单纯点云分类或检测

## 2. 推荐的数据目录结构

建议把数据放成下面这种结构：

```text
3d-adv-lab/
  data/
    kitti/
    nuscenes/
    modelnet40/
    scanobjectnn/
  work_dirs/
  outputs/
    clean/
    adversarial/
    defended/
    vis/
```

含义如下：

- `data/`：原始数据集
- `work_dirs/`：训练或推理输出
- `outputs/clean/`：干净样本结果
- `outputs/adversarial/`：攻击样本结果
- `outputs/defended/`：防御样本结果
- `outputs/vis/`：最终可视化图像或点云渲染结果

## 3. 如何把数据导入项目

### 3.1 最稳妥的做法：使用数据根目录配置

把数据集下载后放到 `data/` 目录下，然后在 MMDetection3D 配置文件里指定 `data_root`。

例如：

```python
data_root = 'data/nuscenes/'
```

### 3.2 推荐用符号链接或固定路径

如果你们多人协作，最好统一一个绝对路径或固定相对路径，避免不同电脑上配置不一致。

### 3.3 数据导入检查

导入后至少检查三件事：

- 目录是否完整
- 标注文件是否存在
- 训练/验证/测试划分是否能正常读取

## 4. 干净样本怎么获得

干净样本就是原始数据，没有攻击、没有扰动。

建议生成方式：

1. 从数据集里抽取一小批样本
2. 直接走模型推理
3. 保存原始输入和模型输出

建议保存内容：

- 原始点云文件
- 原始图像文件，如果是多模态任务
- 模型预测结果
- 评估指标

干净样本是后面所有对比实验的基准。

## 5. 攻击样本怎么获得

建议先从低风险、容易实现的输入级攻击开始，不要一开始就做复杂物理攻击。

### 5.1 点云攻击的起步方式

可以先做这些扰动：

- 高斯噪声
- 点抖动
- 点删减
- 点插入
- 局部遮挡
- 坐标平移
- 旋转扰动

这些方式适合先验证模型对点云变化是否敏感。

### 5.2 攻击样本生成流程

推荐流程如下：

1. 读取原始点云
2. 按攻击策略生成扰动点云
3. 把扰动后的样本单独保存
4. 用模型对攻击样本做推理
5. 统计性能变化

### 5.3 攻击样本保存建议

建议把攻击样本单独存到：

```text
outputs/adversarial/
```

并记录：

- 攻击类型
- 攻击强度
- 样本编号
- 模型输出
- 攻击前后性能差异

## 6. 防御样本怎么获得

防御样本不是“重新生成的一类数据”，而是“经过防御处理后的输入”或“经过防御模型处理后的输出”。

常见做法有两种：

### 6.1 输入净化型防御

先对攻击样本做处理，再送入模型，例如：

- 去噪
- 平滑
- 体素化
- 过滤离群点

这类方法得到的是“防御后的输入样本”。

### 6.2 鲁棒模型型防御

对抗训练后的模型直接处理攻击样本，这时防御结果体现在模型输出上。

### 6.3 防御样本保存建议

建议把防御结果存到：

```text
outputs/defended/
```

保存：

- 防御前的攻击样本
- 防御后的输入或中间结果
- 防御后的模型输出

## 7. 如何得到三类样本

建议你们固定同一批测试 ID，然后对每个样本生成三份结果：

- 干净样本：原始输入
- 攻击样本：原始输入经过攻击变换
- 防御样本：攻击样本经过防御处理，或由鲁棒模型输出

这样最方便做对比表。

推荐的对应关系是：

| 样本类型 | 输入 | 输出 |
| :--- | :--- | :--- |
| 干净样本 | 原始数据 | 基线预测 |
| 攻击样本 | 攻击后的数据 | 攻击预测 |
| 防御样本 | 攻击后的数据 + 防御 | 防御预测 |

## 8. 可视化结果怎么获得

### 8.1 点云可视化

推荐使用 Open3D 直接渲染点云：

- 原始点云
- 攻击后点云
- 防御后点云

对比时建议统一相机视角和颜色映射。

### 8.2 3D 检测结果可视化

建议把下面内容画出来：

- 3D 点云
- GT 框
- 预测框
- 攻击后预测框
- 防御后预测框

这样可以直观看到：

- 漏检
- 框漂移
- 误分类
- 置信度下降

### 8.3 建议的输出文件

建议保存为：

- `png`：静态截图
- `ply` 或 `pcd`：点云文件
- `json`：预测结果和指标
- `html` 或 notebook 输出：交互式结果

## 9. 最建议的实验顺序

如果你现在就要开始，建议按这个顺序做：

1. 配置数据集路径
2. 跑干净样本推理
3. 保存干净样本可视化
4. 对同一批样本做简单点云攻击
5. 保存攻击样本和攻击后预测
6. 加入一个简单防御处理
7. 保存防御样本和防御后预测
8. 汇总三组指标和三组可视化图

## 10. 推荐的指标对比

### 点云分类

- Accuracy
- Attack Success Rate
- Robustness Drop

### 3D 检测

- mAP
- Recall
- 漏检率
- 框偏移量
- 攻击前后性能下降幅度

## 11. 你现在最应该先做什么

如果目标是尽快出结果，建议先做这三步：

1. 选一个数据集
2. 跑通一个基线模型
3. 做最简单的点云扰动测试

先不要急着把所有攻击算法都加进去，先把“三类样本 + 三组结果 + 三张图”跑出来。
