# 三维点云攻击与防御测试指南

本文用于回答两个问题：当前环境是否已经配置完成，以及配置完成后如何对三维点云做攻击与防御测试。

## 1. 当前是否算配置好

如果你的运行目标是 Ubuntu 或 Docker 容器中的这套环境，那么答案是：**基本已经配置好**。

已经验证通过的核心组件包括：

- ART 1.20.1
- MMCV 2.1.0
- MMEngine 0.10.7
- MMDetection 3.2.0
- MMDetection3D 1.4.0
- Open3D 0.19.0
- PyTorch 2.11.0+cu130

同时，导入检查已经通过，说明最基本的依赖闭环可用。

但需要注意：

- 这不等于已经完成完整项目开发
- 还需要你们自己的模型配置、数据集和实验脚本
- 若要真正做 GPU 训练，还需要宿主机 NVIDIA 驱动和容器 GPU 直通正常

## 2. 推荐测试顺序

建议按下面顺序做，不要一上来就直接跑完整攻击实验。

1. 环境导入测试
2. 数据集加载测试
3. 基线模型推理测试
4. 攻击测试
5. 防御测试
6. 统一评估和对比

## 3. 最小验证闭环

先确认下面三件事都正常：

- `torch` 能导入
- `mmdet3d` 能导入
- 你选定的数据集能被正确读取

建议先从小规模数据开始，比如：

- ModelNet40，用于分类类点云测试
- KITTI 或 nuScenes，用于 3D 检测测试

## 4. 点云攻击测试怎么做

### 4.1 先做输入级扰动

最稳妥的起点不是直接做复杂对抗算法，而是先做输入级攻击或扰动，包括：

- 随机噪声扰动
- 点抖动（jitter）
- 点稀疏化（dropout）
- 坐标偏移
- 局部遮挡

这样可以先验证模型是否对点云扰动敏感。

### 4.2 再做对抗攻击

如果你们要做更规范的攻击实验，可以考虑：

- 基于梯度的扰动
- 点插入
- 点删除
- 邻域约束攻击

对于分类任务，可以先用 ART 的标准接口验证攻击流程；
对于 3D 检测任务，更常见的做法是先在 MMDetection3D 的输入管线层做点云扰动，再统计性能变化。

### 4.3 攻击测试输出什么

建议记录这些结果：

- 原始性能
- 攻击后性能
- 性能下降幅度
- 攻击成功率
- 不同扰动强度下的曲线

## 5. 防御测试怎么做

防御测试建议从轻量方案开始，不要一开始就做复杂鲁棒训练。

### 5.1 数据级防御

- 随机采样增强
- 点云去噪
- 体素化平滑
- 训练时加入扰动增强

### 5.2 模型级防御

- 对抗训练
- 噪声注入
- 输入净化
- 简单检测机制

### 5.3 防御测试输出什么

建议记录：

- 防御前后性能差异
- 攻击下的鲁棒性提升
- 额外计算开销
- 对干净样本性能的影响

## 6. 建议的评估指标

不同任务的指标不一样。

### 6.1 分类任务

- Accuracy
- Attack Success Rate
- Robustness Drop

### 6.2 3D 检测任务

- mAP
- Recall
- 攻击下的 mAP 下降量
- 检测框偏移
- 漏检率

## 7. 推荐的实验设计

建议按三组对比做：

1. 干净输入下的基线模型
2. 攻击输入下的模型
3. 攻击输入下加入防御后的模型

这样就能回答三个最关键的问题：

- 模型是否脆弱
- 哪种攻击更有效
- 防御是否真的有效

## 8. 适合你们当前阶段的落地方案

如果你们现在就要开始做，最建议的路线是：

1. 先选一个多模态 3D 检测模型
2. 先跑通干净样本推理
3. 再做简单点云扰动测试
4. 再做一个轻量防御对比
5. 最后再考虑更复杂的对抗攻击

## 11. 最小烟雾测试脚本

工作区已经提供一个可以直接跑的最小脚本：

- [scripts/minimal_pointcloud_smoke_test.py](scripts/minimal_pointcloud_smoke_test.py)

运行命令：

```bash
python scripts/minimal_pointcloud_smoke_test.py --output-dir outputs/smoke_test
```

这个脚本会：

- 检查核心依赖版本
- 生成一份干净点云
- 生成一份攻击点云
- 生成一份防御点云
- 保存报告文件

默认会输出这些结果：

- `clean.ply` 和 `clean.npy`
- `adversarial.ply` 和 `adversarial.npy`
- `defended.ply` 和 `defended.npy`
- `report.json`

如果当前机器缺少图形或系统库，脚本会自动降级，仍然输出 `.npy` 和报告，不会直接中断。
